duży hosting · pełna edycja

GitHub Pages to naprawdę solidny zbiornik na pliki i ruch — daje Ci dużo miejsca, dużo transferu, i co najważniejsze: możliwość edycji strony w dowolnym momencie. Wrzucasz raz, a potem podmieniasz tylko to co chcesz zmienić.

🌐 Netlify Drop

Wrzucasz mapę raz — i koniec. Jak chcesz coś poprawić, musisz wrzucić od nowa pod nowym adresem.

🐙 GitHub Pages

Wrzucasz mapę raz, a potem edytujesz w dowolnym momencie — adres zostaje ten sam, tylko zawartość się zmienia.

1. Załóż konto → 2. Stwórz repo →

// najpierw rejestracja, potem nowe repozytorium

✉️ Krok 1 — Rejestracja

1
Kliknij pomarańczowy przycisk "Załóż konto"

Otworzy się formularz GitHuba. Wypełnij po kolei:

  • Podaj adres e-mail
  • Wymyśl hasło (zapisz je sobie)
  • Wybierz nazwę użytkownika — będzie częścią adresu Twojej strony, więc lepiej coś sensownego (np. jan-kowalski, mapa-okolicy)
  • Potwierdź mail klikając w link który Ci przyślą
Uwaga z nazwą: nazwa użytkownika będzie widoczna w adresie Twojej strony — np. jan-kowalski.github.io/moja-mapa. Wybierz coś, czego nie będziesz się wstydził za 10 lat 🙂

📦 Krok 2 — Utwórz repozytorium

Repozytorium (skrót: "repo") to po prostu folder w internecie — to tam wrzucisz swoją mapę. Każda mapa = osobne repo.

1
Kliknij fioletowy przycisk "Stwórz repo"

Otworzy się formularz nowego repozytorium. Wypełnij tylko trzy rzeczy:

  • Repository name — nazwa repo, czyli ostatni kawałek adresu Twojej strony. Np. moja-mapa, parafia, szlak-turystyczny
  • Public — zaznacz! Tylko publiczne repo dostają darmowy GitHub Pages
  • Add a README file — zaznacz, żeby repo się utworzyło z gotową strukturą

Resztę pól zostaw bez zmian. Na dole kliknij "Create repository".

Nazwa repo będzie w adresie: jeśli Twoja nazwa użytkownika to jan-kowalski, a repo nazwiesz moja-mapa, to adres strony będzie: jan-kowalski.github.io/moja-mapa/. Wybierz krótko i prosto.

📤 Krok 3 — Wrzucanie plików mapy

1
W swoim repo kliknij "Add file" → "Upload files"

Przycisk "Add file" jest po prawej stronie, nad listą plików. Z rozwijanej listy wybierz "Upload files".

2
Przeciągnij swoje pliki

Na otwartą stronę przeciągnij wszystko co składa się na Twoją mapę:

  • Plik index.html
  • Folder img (ze zdjęciami)
  • Folder audio (z nagraniami)

Możesz też wrzucić cały folder z mapą — GitHub rozpakuje go automatycznie.

3
Zatwierdź wgranie (Commit)

Po wgraniu plików zjedź na dół strony i kliknij zielony przycisk "Commit changes". To zapisuje Twoje pliki w repo.

Słowo "commit": to po prostu "zatwierdź zmiany". Każda zmiana w repo wymaga commitu — taki podpis pod zmianą.

⚙️ Krok 4 — Włącz GitHub Pages

Same pliki w repo to jeszcze nie strona w internecie. Trzeba GitHubowi powiedzieć: "zrób z tego stronę".

1
Wejdź w "Settings" swojego repo

U góry strony repo jest pasek zakładek (Code, Issues, Pull requests...). Na końcu po prawej jest "Settings" — kliknij.

2
W menu po lewej znajdź "Pages"

Otworzy się ustawienia GitHub Pages.

3
Wybierz źródło strony
  • W sekcji Source wybierz: Deploy from a branch
  • W sekcji Branch wybierz: main i folder / (root)
  • Kliknij Save

Gotowe! GitHub zacznie budować Twoją stronę.

🔗 Krok 5 — Gdzie znaleźć adres swojej strony

Po włączeniu GitHub Pages (z poprzedniego kroku), na tej samej stronie ustawień "Pages" pojawi się zielona ramka z Twoim adresem.

Adres będzie wyglądał tak:

// twój adres strony
https://twoja-nazwa-uzytkownika.github.io/nazwa-repo/

Np. jeśli nazwa użytkownika to jan-kowalski, a repo nazwałeś moja-mapa:

// przykład
https://jan-kowalski.github.io/moja-mapa/
Czekaj 1-2 minuty po włączeniu Pages, zanim adres zacznie działać. GitHub musi "zbudować" stronę. Jak otworzysz za szybko, zobaczysz białą stronę albo błąd 404.

⭐ Krok 6 — Bonus: edytuj mapę kiedy chcesz

To jest największa przewaga GitHuba nad Netlify Drop. Możesz w dowolnym momencie zaktualizować mapę bez zmiany adresu strony.

A
Podmień plik index.html

Gdy dodasz nowe punkty na mapie i wygenerujesz świeży index.html:

  • Wejdź do swojego repo na GitHubie
  • Kliknij stary plik index.html
  • Po prawej stronie kliknij ikonę kosza (lub ołówka, jeśli chcesz edytować bezpośrednio)
  • Zatwierdź zmianę przyciskiem "Commit changes"
  • Teraz Add file → Upload files i wrzuć nowy index.html
B
Dodaj nowe zdjęcia lub nagrania

Wchodzisz do odpowiedniego folderu (img lub audio), klikasz Add file → Upload files, przeciągasz nowe pliki, commit. Gotowe.

C
Po 1-2 minutach strona się aktualizuje

GitHub automatycznie odświeży Twoją stronę pod tym samym adresem. Nikt z odwiedzających niczego nie musi robić — wystarczy że odświeży stronę w przeglądarce.

Historia zmian: GitHub przechowuje historię wszystkich Twoich edycji — możesz wrócić do dowolnej poprzedniej wersji, jeśli coś zepsujesz. Bezcenne, gdy popełnisz błąd.

📊 Ile miejsca i transferu dostajesz

GitHub Pages w darmowym planie daje Ci spore zasoby — zwykle aż nadto dla mapy mini-klik:

  • ✦ 1 GB miejsca na pliki w jednym repo — wystarcza na setki zdjęć i nagrań
  • ✦ 100 GB transferu miesięcznie — czyli ruchu od odwiedzających
  • ✦ Nieograniczona liczba repo — możesz mieć wiele map na jednym koncie
  • ✦ Bez rachunku za przekroczenie — w razie czego GitHub po prostu napisze maila
W praktyce: dla średniej mapy o wadze 10 MB, 100 GB transferu starcza na ok. 10 tysięcy wejść miesięcznie. Dla lżejszej mapy (1 MB) — to nawet 100 tysięcy wejść. Spokojnie.

⚠️ Uwaga na prawa autorskie

GitHub — tak jak Netlify — może zablokować Twoje repo jeśli wykryje że łamiesz prawa autorskie.

Czego nie wrzucaj:

  • ✗ Cudzych zdjęć ściągniętych z internetu bez zgody autora
  • ✗ Cudzej muzyki — utworów z YouTube, Spotify, radia
  • ✗ Cudzych nagrań głosowych, podcastów, audiobooków

Bezpieczne są Twoje własne zdjęcia, Twoje własne nagrania, oraz materiały z licencją Creative Commons.
